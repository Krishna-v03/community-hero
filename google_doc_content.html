🦸‍♂️ COMMUNITY HERO

Hyperlocal Problem Solver powered by Google Gemini & Google Cloud

Official Hackathon Submission Project Documentation

1. Executive Summary & Impact Metrics

Community Hero is an autonomous, trust-weighted civic platform designed to connect citizens, field officers, and city administrators to rapidly resolve public infrastructure failures (potholes, garbage pile-ups, broken streetlights, water leaks). By placing Google Gemini AI at the core of report triaging, queue management, and service operations, we transform random citizen complaints into structured, actionable municipal tickets.

Ultimate Value Proposition:
For municipal bodies overwhelmed by disorganized public complaints, Community Hero is a zero-friction, self-verifying triage system that slashes issue verification times, eliminates duplicate tasks automatically using GPS boundaries, and guides field workers through automated scheduling—delivering real-time transparency to citizens.

Key Impact Metrics (Forecasted)

  • 95% Reduction in Triage Delay: AI classifies and assigns uploaded complaints in under 3 seconds, replacing manual queues that took up to 5 business days.
  • 100% Duplicate Prevention: Geolocation checkups prevent duplicate entries of the same issue within 60 meters, routing subsequent reporters to confirm the existing ticket.
  • 75% Shorter On-site Check-in Times: Mobile navigation and geo-boundary validation ensure field officers are routed directly to coordinates, logging on-site attendance instantly.

2. Problem Statement Selected

Urban infrastructure decay (potholes, broken lights, leakages) degrades standard of living and presents hazard risks. Current municipal complaint frameworks fail due to structural backlogs.

A. Root Cause Analysis

Traditional municipal structures rely on manual sorting. Reports come through scattered channels (emails, calls, social media mentions) and lack structured metadata (such as verified GPS coordinates, severity metrics, and category tags). This creates backlogs where critical issues are buried under non-critical complaints or spam.

C. Why Existing Solutions Fail

  • No Geofenced Deduplication: Existing citizen apps allow infinite reports for the same pothole, creating massive noise in databases.
  • No Trust Weighting: All complaints are treated equally. A malicious report has the same weight as a report verified by five local residents.
  • Manual Verification Bottleneck: Cities must deploy field inspectors just to verify if a complaint is real, wasting operational budget.

3. Solution Overview & Architecture

Community Hero resolves these issues by building a unified database with role-based dashboard screens and an intelligent AI layer. The backend automates validation using Google Gemini vision models and spatial databases.

6. Google Technologies Utilized & Cloud Infrastructure

A. Google Gemini AI API

We integrated the official @google/genai SDK, utilizing the gemini-2.5-flash model. This model offers the optimal balance of image comprehension, JSON schema formatting compliance, and rapid response speed. It powers the following workflows:

  • Image Classification: Extracts clean category arrays and flags spam.
  • Smart Assignment Recommender: Analyzes proximity, backlog, and load.
  • SLA Escalation Generator: Writes professional operational memos summarizing overdue issues.

B. Google Cloud Run

Our app is hosted on Google Cloud Run (europe-west1 region). By deploying a lightweight Node container, the app scales automatically with traffic peaks (e.g., in a storm with many potholes reported) and scales to 0 instances when idle, incurring zero resource costs during downtime.

C. Google Cloud Build & Artifact Registry

We established a secure, fully automated CI/CD pipeline. Every commit pushed to the main branch triggers Google Cloud Build, which builds the Docker image using the repository's Dockerfile and deploys it to Cloud Run. The configuration utilizes the following cloudbuild.yaml style steps:

# Example automated build steps triggered on Git push
steps:
  # Build the container image
  - name: 'gcr.io/cloud-builders/docker'
    args: ['build', '-t', 'europe-west1-docker.pkg.dev/$PROJECT_ID/app/community-hero:$COMMIT_SHA', '.']
  # Push the container image to Artifact Registry
  - name: 'gcr.io/cloud-builders/docker'
    args: ['push', 'europe-west1-docker.pkg.dev/$PROJECT_ID/app/community-hero:$COMMIT_SHA']
  # Deploy container image to Cloud Run
  - name: 'gcr.io/google.com/cloudsdktool/cloud-sdk'
    entrypoint: gcloud
    args:
      - 'run'
      - 'deploy'
      - 'community-hero'
      - '--image'
      - 'europe-west1-docker.pkg.dev/$PROJECT_ID/app/community-hero:$COMMIT_SHA'
      - '--region'
      - 'europe-west1'
      - '--platform'
      - 'managed'
      - '--allow-unauthenticated'

7. Future Roadmap & Scalability Plan

To scale Community Hero from a hackathon prototype into a city-wide operational solution, we outline a three-phase expansion:

Phase 1: Field Trials & Municipal Pilots (1–3 Months)

  • Run a localized pilot with 100 volunteer citizens and 5 city field officers.
  • Calibrate the Proximity Trust-Weighting model based on GPS precision variance in dense high-rise areas.

Phase 2: Advanced AI Predicative Dispatch (3–6 Months)

  • Integrate historical weather forecasts and infrastructure age datasets.
  • Use Gemini models to proactively prompt municipal crews to inspect older streetlights before a major rainstorm occurs.

Phase 3: Multi-City Federated Deployment (6+ Months)

  • Abstract the database structure to support separate municipal logins (multi-tenant architecture).
  • Optimize the PWA to allow citizens to seamlessly switch municipal jurisdictions when traveling across borders.
